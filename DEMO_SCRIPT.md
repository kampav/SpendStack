---

## 0–2 min | Setup & Context

**Say:** "We're going to build and ship a production feature — live, on a real cloud URL — in 40 minutes using Claude at every stage of the SDLC. Here's the pipeline."

## 2–6 min | Research in Claude.ai

**Say:** "Every feature starts with research. I'm going to ask Claude to map the competitive landscape so our design decisions are grounded in what real users hate about existing apps."

**Paste this prompt into Claude.ai:**
```
You are a senior product strategist at a UK retail bank.

Research the current state of UK loyalty and rewards apps (Barclays Rewards, Lloyds Everyday Offers, Nectar, Tesco Clubcard, Monzo Perks). For each:
- What does it do well?
- What are the top 3 user complaints (based on App Store reviews, Reddit, and general UX patterns)?
- What engagement mechanic does it use (if any)?

Then identify the 3 biggest unmet needs across all of them that a gamified household rewards app could solve.

Format as a structured briefing, not bullet soup.
```

**Expected output:** Claude produces a structured competitive brief covering 5 apps, with an unmet needs section that maps to SpendStack's features (streaks, household competition, AI personalisation).

**Fallback:** If Claude.ai is slow, read out pre-prepared summary: "The big gaps are: no household social layer, no behavioural streaks, and AI-powered personalisation is completely absent across all incumbents."

## 6–12 min | Design Mockup in Claude.ai Artifact

**Say:** "Now let's use Claude to generate the dashboard design so the engineering team has a visual north star before a single line of code is written."

**Paste this prompt into Claude.ai:**
```
Create an interactive HTML/CSS mockup of the SpendStack dashboard — a gamified UK retail banking loyalty app.

Design system:
- Primary green: #006A4D, Accent: #38A169, Navy: #1A2B3C, Gold: #D4A017, Cream background: #FBF9F4
- Font: Inter. Rounded corners (12px). Premium UK banking feel — not playful.
- Three pre-built features must be visible:
  1. Household Leaderboard — ranked rows with name, avatar initials, points, tier badge
  2. Streak Tracker — a 30-day calendar grid with active (green), missed (grey), today (gold ring) states
  3. Tier Progression — a progress bar from current tier to next, with tier name and badge

Layout: mobile-first single column. Top: greeting + tier badge. Then: TierMeter card. Then: StreakCalendar card. Then: top 3 leaderboard rows with "See all →".

Seed with fictional UK household data: 3 members — Priya (4,200 pts, Gold, 12-day streak), James (3,100 pts, Silver, 5-day streak), Cleo (1,800 pts, Silver, 2-day streak). Priya is highlighted as "you".

Make it look real — not a wireframe.
```

**Expected output:** Claude renders an interactive HTML artifact showing the SpendStack dashboard with all three features, real colours, and fictional data.

**Fallback:** If artifact renders slowly, narrate: "Claude is generating a pixel-accurate mockup — this would normally take a designer a day. We'll use this as our visual spec for the build." Switch to pre-saved screenshot if needed.

### 14–18 min | Feature 1: Household Leaderboard

**Say:** "First feature: the leaderboard. I'm giving Claude a single prompt, and it will write the component, the Firestore hook, seed data, and tests."

**Paste:**
```
Read CLAUDE.md, PRD.md, and DESIGN.md first.

Build the Household Leaderboard feature:
- LeaderboardRow component: rank, Avatar (initials fallback), displayName, tier badge, monthly points (right-aligned primary green), weekly delta (caption)
- LeaderboardList component: takes sorted User[] prop, highlights own uid row with left border and 4% primary green background
- useLeaderboard hook: real-time Firestore listener on /users ordered by currentMonthPoints desc, filtered by householdId
- Seed 3 fictional users in a dev Firestore emulator script
- Match DESIGN.md exactly: colours, spacing, typography, accessibility (aria-labels, role="list")
- Add Vitest unit tests for LeaderboardRow rendering and ranking logic
- Run tests, fix any failures, then commit: feat: add household leaderboard
```

**Expected output:** Claude Code creates `LeaderboardRow.tsx`, `LeaderboardList.tsx`, `useLeaderboard.ts`, seed script, tests. Tests pass. Commit visible.

**Fallback:** If Claude Code hangs > 90 seconds, Ctrl+C and re-run with: "Continue from where you left off. Focus on LeaderboardRow.tsx first, then the hook."

### 18–22 min | Feature 2: Streak Tracker

**Say:** "Feature two: the streak tracker. Thirty days of spend history, visualised as a calendar."

**Paste:**
```
Read CLAUDE.md and DESIGN.md.

Build the Streak Tracker feature:
- StreakCalendar component: 7-column CSS grid, 30 cells, each 36px circle
  - States: active (accent green fill, white check icon), missed (neutral-200), today-active (accent + gold ring), today-pending (dashed border), future (empty)
  - Milestone star markers below days 7, 14, 30
  - aria-label on each cell: e.g. "Monday 21 April, streak active"
- useStreak hook: reads /streaks/{uid} from Firestore, returns activeDays[], currentStreak, longestStreak
- streak/engine.ts: pure functions — isQualifyingDay(), updateStreak(), detectMilestone()
- Vitest tests for engine.ts: test streak increment, reset on gap, milestone at 7 days
- Match DESIGN.md exactly
- Run tests, commit: feat: add streak tracker
```

**Expected output:** StreakCalendar renders with correct states. Engine tests pass (at least 5 cases). Commit made.

**Fallback:** "Claude, the calendar grid cells aren't showing the gold ring for today. Fix only the today-active CSS state and re-run tests."

### 22–26 min | Feature 3: Tier Progression

**Say:** "Last pre-built feature: the tier meter. This is what makes the loyalty loop feel real — customers always know how close they are to the next level."

**Paste:**
```
Read CLAUDE.md and DESIGN.md.

Build the Tier Progression feature:
- TierMeter component: horizontal layout — tier badge (left), ProgressBar (centre), "X pts to Gold" label (right)
  - Four tiers: Everyday (0–999), Silver (1000–4999), Gold (5000–14999), Platinum (15000+)
  - Badge colours from DESIGN.md Badge component spec
  - Platinum: shimmer animation on badge, "Maximum tier achieved" text, no progress bar
  - ProgressBar: animated fill on mount (600ms ease-out), aria-valuenow/min/max
- useTier hook: derives tier from user.lifetimePoints, calculates progress percentage and points gap
- tier/resolver.ts: pure function resolveTier(points: number): Tier
- Vitest tests: boundary conditions — 999, 1000, 4999, 5000, 14999, 15000
- Run tests, commit: feat: add tier progression
```

**Expected output:** TierMeter renders correctly across all four states. Boundary tests pass. Commit made.

**Fallback:** "Skip the Platinum shimmer animation for now — just show the badge and static text. Keep the tests passing."

## Recovery Plays

**If Claude Code hangs (> 90 seconds with no output):**
Ctrl+C. Type: "Continue your previous task. Start with the file you were last editing and complete it. Then move to the next file." If it hangs again, kill the session, reopen Claude Code, and paste a scoped version of the prompt targeting one file only.

**If GitHub Actions fails:**
Click the failing job, read the first error line aloud. Say: "This is CI doing its job — it caught an issue before it reached users." If it's a type error: paste the error into Claude Code with "Fix this TypeScript error and push the fix." If it's an env var issue: add the secret in GitHub Settings → Secrets and re-run the job.

**If Cloud Run deploy errors (non-502):**
`gcloud run services describe spendstack --region europe-west2` — read the error. Most common cause: missing env var. Add via `gcloud run services update spendstack --set-env-vars KEY=VALUE`. Say: "Cloud Run returns detailed logs — no mystery debugging."

**If Wi-Fi drops:**
Switch to mobile hotspot immediately (pre-tested, already in iPhone Settings → Hotspot ready). If hotspot also fails: switch to localhost demo (pre-seeded, `npm run dev` already running). Say: "Offline-capable demo — the app and Claude Code both work locally." Play backup video only if localhost has an unexpected error.
