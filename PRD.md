# PRD.md — SpendStack Product Requirements

**Vision**
SpendStack transforms everyday UK retail banking spend into a competitive household sport — turning passive transactions into points, streaks, and tier progression that the whole family can see and chase. It gives retail banking customers a reason to consolidate spend on their primary bank account, deepening engagement and reducing churn.

**Target User**
UK retail banking customers aged 25–45, in multi-person households, with at least one active debit or credit card. Likely parents, couples sharing finances, or flatmates — motivated by value, fairness, and light competition.

**Problem Statement**
UK banks offer cashback and rewards but almost none surface them in a way that changes behaviour. Points expire silently. Customers don't know their tier. Nobody in the household knows who's contributing most. SpendStack makes rewards visible, social, and sticky.

---

## Features

### 1. Household Leaderboard (Pre-built)

**User story:** As a household member, I want to see a ranked list of everyone's points this month so I know where I stand and feel motivated to keep spending.

**Acceptance criteria:**
- Leaderboard shows all household members ranked by points earned in the current calendar month
- Each row shows: rank, avatar, display name, points total, delta vs previous week
- Updates in real time via Firestore listener
- My own row is visually highlighted
- If household has 1 member, prompt to invite others is shown instead of a leaderboard

---

### 2. Streak Tracker (Pre-built)

**User story:** As a user, I want to see my consecutive days with qualifying spend so I'm rewarded for consistent engagement.

**Acceptance criteria:**
- Streak shows the current consecutive-day count of qualifying transactions
- Calendar view shows last 30 days, colour-coded: active (green), missed (muted), today (gold ring)
- A qualifying transaction is any debit/credit of ≥ £1.00 on a linked card
- Missing one day resets the streak to 0
- Streak milestone badges at 7, 14, 30, 60 days
- Bonus points awarded automatically at milestones (100pts / 250pts / 500pts / 1000pts)

---

### 3. Tier Progression (Pre-built)

**User story:** As a user, I want to know my current loyalty tier and how close I am to the next one so I feel rewarded for cumulative spend.

**Acceptance criteria:**
- Four tiers: Everyday (0–999 pts), Silver (1,000–4,999), Gold (5,000–14,999), Platinum (15,000+)
- Tier based on cumulative lifetime PointsLedger balance
- TierMeter shows: current tier badge, points to next tier, progress bar
- Tier downgrades do not apply in MVP (tiers are permanent once earned)
- Tier badge shown on leaderboard rows and profile header

---

### 4. AI Quests (Live-add candidate)

**User story:** As a user, I want personalised spending challenges generated by AI so I have a reason to try new spend categories.

**Acceptance criteria:**
- Claude generates 3 quests per user per week based on recent transaction history
- Quest format: title, description, target category, spend threshold, points reward, expiry date
- Quests are displayed on a dedicated Quests screen as cards
- Completing a quest (met by transaction data) marks it done and credits points
- Expired incomplete quests are archived, not deleted
- Quest generation prompt includes last 30 days of transactions — never raw PII

---

### 5. Redemption Store (Live-add candidate)

**User story:** As a user, I want to redeem my points for real rewards so the points feel valuable.

**Acceptance criteria:**
- Store shows a curated catalogue: gift cards, cashback credits, charity donations
- Each item shows: image, title, points cost, availability status
- User can filter by category
- Redemption requires confirmation modal ("Spend 2,500 points?")
- Redeemed items create a PointsLedger entry with negative points
- Redemption history shown on Profile screen
- Balance cannot go below zero (UI disables button if insufficient points)

---

### 6. AI Spending Coach (Live-add candidate)

**User story:** As a user, I want AI commentary on my spending patterns so I can make better financial decisions and earn more points.

**Acceptance criteria:**
- Coach screen shows a scrollable conversation-style interface
- On load, Claude generates a weekly summary: top spend categories, streak status, tier gap
- User can ask follow-up questions; responses stream in character-by-character
- System prompt instructs Claude to act as an encouraging UK retail banking coach — never give regulated financial advice
- All requests are server-side; Anthropic API key never reaches the client
- Responses must be ≤ 250 tokens for MVP; truncated gracefully with "See more" if needed

## Data Model

### User
| Field | Type | Notes |
|---|---|---|
| `uid` | string | Firebase Auth UID, document ID |
| `displayName` | string | Shown on leaderboard and profile |
| `email` | string | Auth-linked |
| `avatarUrl` | string | URL to avatar image or initials fallback |
| `householdId` | string | FK → Household.id |
| `tierId` | string | Current tier: everyday / silver / gold / platinum |
| `lifetimePoints` | number | Running total for tier calculation |
| `currentStreak` | number | Consecutive qualifying days |
| `longestStreak` | number | All-time max |

| `createdAt` | timestamp | |

## Success Metrics

| Metric | Target (90 days post-launch) |
|---|---|
| DAU/MAU | ≥ 35% |
| Household invite rate | ≥ 40% of users invite at least one member |
| Streak 7-day achievement | ≥ 25% of active users |
| Quest completion rate | ≥ 50% of generated quests completed |
| Redemption conversion | ≥ 15% of users redeem within 30 days |
| Coach engagement | ≥ 3 messages/user/week on adopters |
